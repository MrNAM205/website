## üöÄ Getting Started

### 1. Fork the Repository

Fork the `Sovereign-Invocation-Cockpit` repository to your GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/Sovereign-Invocation-Cockpit.git
cd Sovereign-Invocation-Cockpit
```

### 3. Install Dependencies

This project uses Node.js and npm.

```bash
npm install
```

### 4. Set Up Environment Variables

Create a `.env.local` file in the root of the project and add your Gemini API key:

```
GEMINI_API_KEY=YOUR_GEMINI_API_KEY
```

### 5. Run the Development Server

```bash
npm run dev
```

This will start the development server, usually at `http://localhost:5173`.

## üß¨ Code Standards & Guidelines

- **TypeScript First:** All new code should be written in TypeScript.
- **React Best Practices:** Follow modern React functional component patterns, hooks, and context API where appropriate.
- **Tailwind CSS:** Utilize Tailwind CSS for styling. Maintain consistency with existing utility classes.
- **Semantic Naming:** Use clear, descriptive names for variables, functions, and components that reflect their purpose within the sovereign framework.
- **Comments:** Add comments for complex logic or philosophical context, explaining *why* a decision was made.
- **Testing:** Write unit and integration tests for new features and bug fixes.

## üí° Submitting Contributions

### 1. Create a New Branch

```bash
git checkout -b feature/your-feature-name
```

### 2. Implement Your Changes

Write your code, ensuring it adheres to the code standards and guidelines.

### 3. Write Tests

Ensure your changes are covered by appropriate tests.

### 4. Commit Your Changes

Write clear, concise commit messages that explain the *what* and *why* of your changes.

```bash
git commit -m "feat: Add [Feature Name] for [Purpose]"
```

### 5. Push to Your Fork

```bash
git push origin feature/your-feature-name
```

### 6. Create a Pull Request

Open a pull request from your fork to the `main` branch of the upstream repository. Provide a detailed description of your changes and their alignment with the project's philosophy.
